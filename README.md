# StoryGuide

## Тема проекта
Кино (Советчик)

## Описание проекта
`StoryGuide` — интеллектуальный советчик, который рекомендует похожие фильмы по интересам пользователя и умеет анализировать визуальный контент по теме кино.

Основной сценарий: пользователь пишет, что ему нравится (например, «Мне нравится Матрица»), а система предлагает близкие варианты (например, «Начало»).

## Задачи
1. Фильтры:
- Правила по году и жанру.
2. Анализ сюжета:
- Поиск похожих фильмов по смыслу описания.
3. Жанр по постеру:
- Классификация: драма или ужасы.
4. Computer Vision:
- Загрузка постера фильма или изображения с текстом о фильме.
- OCR через EasyOCR и предобработка через OpenCV.
- Извлечение названия, жанровых подсказок и формирование запроса для рекомендаций.

## Цель
Подбирать похожие фильмы с помощью правил и векторного поиска, а также извлекать данные из изображений.

## Датасеты
1. IMDB Dataset of 50K Movie Reviews:
- https://kaggle.com/datasets/lakshmi25npathi/imdb-dataset-of-50k-movie-reviews
2. Movie Genre from its Poster:
- https://kaggle.com/datasets/neha1703/movie-genre-from-its-poster
3. The Movies Dataset:
- https://www.kaggle.com/datasets/rounakbanik/the-movies-dataset
4. MovieLens 25M:
- https://grouplens.org/datasets/movielens/25m/

Датасеты подключены в рабочую логику проекта:
- `data/raw/MovieGenre.csv` используется как основной каталог фильмов, жанров и рейтингов для поиска и рекомендаций.
- `data/raw/IMDB Dataset.csv` используется для анализа тональности англоязычных отзывов и текстовых вложений.
- `data/raw/movies_metadata.csv`, `credits.csv`, `keywords.csv` обогащают фильмы описаниями, ключевыми словами, актерами и режиссерами.
- `data/raw/ml-25m/...` добавляет пользовательские рейтинги, теги и связи `movieId -> imdbId/tmdbId`.
- `data/raw/FullMoviePosters/...` используется для визуального сопоставления постеров с фильмами из каталога.

StoryGuide работает в fail-fast режиме:
- runtime fallback-ветки и заглушки удалены;
- обязательные данные должны лежать в `data/raw` в полном составе;
- при отсутствии зависимости, индекса или ключевого датасета приложение сообщает явную ошибку, а не продолжает работу в урезанном режиме.

## Интеграция
- Векторный поиск (`Embeddings`) для подбора похожих рекомендаций.
- OpenCV для предобработки постеров и страниц с описанием фильмов.
- EasyOCR для распознавания названий фильмов и связанного текста.
- `MovieGenre.csv` как базовый каталог фильмов.
- `The Movies Dataset` для `overview`, `tagline`, `keywords`, `cast`, `director`.
- `MovieLens 25M` для `ratings`, `tags`, `movieId/imdbId/tmdbId`.
- `FullMoviePosters` для визуального распознавания загруженного постера по самому изображению.
- `IMDB Dataset.csv` для анализа тональности отзывов.
- `data/processed/merged_movie_metadata.pkl` как объединенный кэш метаданных.
- `data/processed/text_search_index.pkl` как текстовый индекс по `overview + keywords + cast + director + tags`.
- `data/processed/poster_index.pkl` как визуальный индекс постеров.

Основной recommendation flow идет через объединенный каталог фильмов, собранный из `MovieGenre.csv`, `The Movies Dataset` и `MovieLens 25M`, без legacy-графовой логики.

## Неделя 7. Computer Vision
- Тема: OCR и классификация изображений.
- Реализация: OpenCV + EasyOCR без fallback-режима.
- Поддерживаемые типы входа: постер фильма, изображение с текстом о фильме.
- Результат: система обрабатывает не только текстовые запросы, но и визуальный контент по теме кино, включая сопоставление постера с фильмом из датасета.

## Неделя 8. Гибридные методы и рекомендации
- Тема: векторные представления и метрики близости.
- Реализация: гибридный поиск `Cosine Similarity + Fuzzy matching`.
- Сценарий под тему проекта: найти фильм по описанию сюжета, неточному названию или смешанному запросу.
- Результат: StoryGuide умеет подбирать похожие фильмы не только по явному совпадению названия, но и по смыслу описания.

## Неделя 9-10. Интеграция модулей
- Тема: объединение символьного и нейросетевого подходов.
- Реализация: единый orchestration pipeline `Вход -> NLP/CV -> Правила -> Решение`.
- Что интегрировано:
- `src/integration_pipeline.py` объединяет текстовый запрос, OCR/visual poster matching и текст из вложений.
- `src/logic.py` выделяет сущности, фильтр по годам, тональность отзывов и строит итоговое решение.
- Rule-based слой дополнительно фильтрует рекомендации по базовым ограничениям каталога из `rules.json`.
- UI теперь запускает не отдельные разрозненные функции, а единый mid-term pipeline и показывает его шаги в ответе.
- Результат: StoryGuide обрабатывает смешанный ввод и выдает одно согласованное решение с трассировкой этапов.

## Неделя 11. Проектирование API и бэкенда
- Тема: модульная архитектура Python.
- Реализация:
- `src/app_service.py` вынесен как service/backend-слой для `Streamlit`.
- `src/project_overview.py` собирает backend-сводку проекта, команды запуска и эксплуатационные метрики.
- `src/ui_presenter.py` готовит таблицы и данные для графиков вне `main.py`, чтобы UI не содержал вычислительной логики.
- UI больше не хранит бизнес-логику чата внутри `main.py`: история, сессии, сохранение состояния и submit-цикл идут через backend-функции.
- `src/integration_pipeline.py` теперь возвращает не только текст ответа, но и структурированный `ui_payload` для интерфейса.
- Результат: логика, хранение состояния и представление разделены по слоям.

## Неделя 12. Пользовательский интерфейс
- Тема: быстрая разработка data apps через `Streamlit`.
- Реализация:
- чат с `chat_input`, встроенной кнопкой прикрепления файлов и кнопкой отправки-стрелкой;
- sidebar с текущим контекстом диалога;
- кнопки `Сбросить фильтры` и `Новая тема`;
- визуальный блок `Последний результат` с метриками, таблицей рекомендаций, графиками по рейтингам, жанрам и годам, а также диагностикой последнего запроса.
- Результат: проект стал полноценным пользовательским приложением, а не только набором функций.

## Неделя 13. Тестирование и подготовка к деплою
- Тема: QA и воспроизводимость.
- Реализация:
- `requirements.txt` зафиксирован по версиям для Python 3.13;
- unit-тесты покрывают chat-flow, backend/service-слой, presenter-логику и структурированный payload для UI;
- проверка запуска выполняется командами `python -m unittest discover -s tests -v` и `streamlit run app.py`.
- Результат: проект воспроизводимо поднимается на чистом окружении и имеет базовую регрессионную защиту.

Ограничение текущих датасетов:
- Локальный индекс покрывает `28847` постеров из `39515` фильмов каталога.
- Текстовый индекс покрывает `40108` фильмов и `1594094` токенов/признаков поиска.
- Постеры покрывают именно афиши, а не кадры из фильма.
- Поэтому постер распознать можно по самому изображению, а произвольный скриншот из фильма надежно определить нельзя без отдельного датасета кадров или обученной visual model.
